# OBD2 App Data Migration to .obd Directory - Implementation Summary

## Overview
Successfully implemented migration of all internal app data from app-private storage to a `.obd` directory within the user-selected log folder. This enables data persistence across install/uninstall cycles during development.

## Implementation Completed

### 1. New Storage Helper Classes

**AppDataDirectory.kt** (`com.sj.obd2app.storage`)
- Manages access to `.obd` directory within user-selected folder
- Provides DocumentFile-based access for SAF compatibility
- Methods for accessing profile files, layout files, settings file, and PID files
- Automatic fallback to app-private storage if no folder selected

**DataMigration.kt** (`com.sj.obd2app.storage`)
- One-time migration coordinator triggered on app startup
- Migrates data from SharedPreferences and app-private files to .obd directory
- Migration flag prevents repeated migrations
- Logs migration progress for debugging

### 2. Updated Data Repositories

**AppSettings.kt**
- Now supports both settings.json (external) and SharedPreferences (fallback)
- In-memory caching for performance
- Consolidated settings including:
  - Active profile ID
  - Default layout name
  - Global polling/command delays
  - Logging flags
  - Auto-connect settings
  - Last connected device info
- Exception: `log_folder_uri` stays in SharedPreferences (bootstrap requirement)

**VehicleProfileRepository.kt**
- Supports individual JSON files per profile: `.obd/profiles/<uuid>.json`
- Fallback to SharedPreferences array format
- Methods updated: `getAll()`, `save()`, `delete()`

**LayoutRepository.kt**
- Supports `.obd/layouts/<name>.json` directory
- Fallback to `context.filesDir/layouts`
- Default layout name moved to AppSettings
- Methods updated: `saveLayout()`, `getSavedLayouts()`, `deleteLayout()`

**PidAvailabilityStore.kt**
- Supports per-profile PID files: `.obd/profiles/<uuid>_pids.json`
- Fallback to SharedPreferences
- Methods updated: `getKnownPids()`, `getLastValues()`, `update()`, `clear()`

**ConnectViewModel.kt**
- Replaced direct SharedPreferences access with AppSettings methods
- Uses `AppSettings.setLastDevice()`, `getLastDeviceMac()`, `getLastDeviceName()`

### 3. Migration Trigger

**MainActivity.kt**
- Added `DataMigration.migrateIfNeeded(this)` in `onCreate()`
- Runs before any other initialization
- Silent migration with logging for debugging

## Directory Structure

```
<selected_folder>/.obd/
├── profiles/
│   ├── <uuid1>.json          # Individual vehicle profile
│   ├── <uuid2>.json
│   ├── <uuid1>_pids.json     # PID availability for profile 1
│   └── <uuid2>_pids.json     # PID availability for profile 2
├── layouts/
│   ├── Default.json          # Dashboard layout
│   ├── Highway.json
│   └── City.json
└── settings.json              # Common app settings
```

## Migration Behavior

1. **On First Launch (No Folder Selected)**
   - App uses SharedPreferences and app-private storage (existing behavior)
   - No migration occurs

2. **After Folder Selection**
   - On next app startup, migration automatically runs
   - All data copied from old locations to `.obd` directory
   - Migration flag set to prevent re-running

3. **Subsequent Launches**
   - App reads/writes directly to `.obd` directory
   - Data persists across app uninstall/reinstall
   - SharedPreferences still used as fallback if folder becomes unavailable

## Testing Recommendations

1. **Fresh Install Test**
   - Install app
   - Create profiles, layouts, adjust settings
   - Select log folder
   - Restart app
   - Verify data persists

2. **Migration Test**
   - Install app with existing data
   - Select log folder
   - Restart app
   - Verify all data migrated to `.obd` directory
   - Check logs for migration success

3. **Uninstall/Reinstall Test**
   - With log folder selected and data in `.obd`
   - Uninstall app
   - Reinstall app
   - Select same log folder
   - Verify data is restored

4. **Fallback Test**
   - Revoke folder permissions
   - Verify app falls back to SharedPreferences
   - Re-grant permissions
   - Verify app switches back to `.obd` directory

## Notes

- All changes maintain backward compatibility
- No data loss during migration
- Migration is idempotent (safe to run multiple times)
- Log folder URI remains in SharedPreferences for bootstrap
- In-memory caching minimizes performance impact
